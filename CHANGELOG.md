## [Unreleased]

### Added
- **VDR-B4 — Dirty-rect tracking** (`pm_types.rs`, `sdl2_renderer.rs`) — `WindowManager::dirty: HashSet<u32>` tracks which HWNDs need recompositing; `composite_and_present()` returns immediately when the set is empty and no drag/resize is active, eliminating unnecessary 60Hz full-desktop blits when nothing has changed. ## [0.0.1] - 2026-04-03

### Fixed
- **OS/2 resource type constants** (`lx/header.rs`) — RT_* constants were using Windows numbering instead of OS/2 numbering; corrected to `RT_MENU=3`, `RT_DIALOG=4`, `RT_STRING=5`, `RT_ACCELTABLE=8`. Previously `WinLoadMenu` could never find the RT_MENU resource in a compiled LX executable because the wrong type ID was passed to the resource manager lookup.
- **MIS_* menu item style constants** (`constants.rs`) — corrected to match OS/2 Warp 4 Toolkit `<pmwin.h>`: `MIS_SEPARATOR=0x0004` (was 0x0020), `MIS_OWNERDRAW=0x0008` (was 0x0800), `MIS_MULTMENU=0x0020`, `MIS_SYSCOMMAND=0x0040` (was 0x0100), `MIS_HELP=0x0080` (was 0x0200), `MIS_STATIC=0x0100` (was 0x0400); added `MIS_BUTTONSEPARATOR`, `MIS_BREAK`, `MIS_BREAKSEPARATOR`, `MIS_GROUP`, `MIS_SINGLE`.
- **DLGTEMPLATE parser rewrite** (`pm_win.rs`) — `parse_dlg_template()` rewritten to match the actual wrc OS/2 DLGTEMPLATE binary format: 10-byte template header (u32 total_size, u16 codepage, u16 offLastItem, u16 cTopItems) at bytes 0–9; child_count at byte 16; title_off at byte 24; dialog position i16 x/y/cx/cy at bytes 30–39; child items from byte 50, each with 10-byte pre-header (class ordinal at [0], text_off at [4]) + 10-byte position block; variable-length data skipped by scanning for the next valid class ordinal. Previously dialogs with multiple control items parsed garbage positions, causing all controls to pile up at (0,0).
- **VDR crash prevention** — `draw_filled_rect()` in `sdl2_renderer.rs` now clamps the destination rectangle to the desktop buffer bounds before writing pixels, preventing an out-of-bounds write that crashed when a dialog's FrameBuffer was smaller than expected; `OS2Window::visible` is now set `true` in both `WinCreateStdWindow` and `create_dialog_from_template()` so newly created windows participate in compositing immediately; dialog OK button geometry corrected by fixing `FrameBuffer` width/height initialisation.
- **MENUTEMPLATE parser** (`pm_win.rs`) — Open Watcom wrc (OS/2 mode) generates a **count-based** 10-byte header (`u32 total_size + u16 codepage + u16 reserved + u16 item_count`) rather than the 8-byte `version+codepage+reserved+cbInfo` format that was assumed; items start at byte 10 (not 8+cbInfo=12). Each SUBMENU item is followed by the same 10-byte child-block header with `child_count`; `parse_menu_items` now reads the sub-header and recurses with the exact child count. MIS_END fallback retained for hand-built inline templates with count=0. Added `test_win_load_menu_submenu` unit test verifying the recursive SUBMENU structure. Previously `WinLoadMenu` parsed garbage (4 top-level items instead of 1 SUBMENU with 3 children).

### Added
- **Virtual Desktop Renderer (VDR) — complete single-surface desktop compositor** (VDR-A1–A5/B1–B3/C1–C5/D1–D6/E1–E3):
  - **VDR-A: Single-surface rendering** — `Sdl2Renderer` replaced per-handle SDL2 windows with one 1024×768 desktop window (`WARPINE_DESKTOP_W/H` env vars); `FrameBuffer { buffer: Vec<u32>, width, height }` per PM handle; `composite_and_present()` blits all visible z-ordered frames bottom-to-top onto the teal `DESKTOP_BG` surface; dialogs use lazy FrameBuffer allocation on first `WM_PAINT` (VDR-A5).
  - **VDR-B: Z-order and focus** — `WindowManager::z_order: Vec<u32>` (back-to-front) + `focused_hwnd: u32`; `z_push_top/bottom`, `z_insert_behind`, `z_remove`, `z_hit_test` helpers; `WinSetActiveWindow` (ord 795) / `WinQueryActiveWindow` (ord 797); `HWND_TOP/BOTTOM/FLOAT/DESKTOP` constants; 60Hz compositor loop via `Duration::from_millis(16)` sleep.
  - **VDR-C: Window chrome** — title bar (navy `0x000080` focused / dark-gray `0x404040` inactive), white title text, close button (VDR-C1); 2-px gray border for FCF_BORDER/FCF_DLGBORDER/FCF_SIZEBORDER (VDR-C2); system-menu icon (small gray square at title-bar left) for FCF_SYSMENU (VDR-C3); close/max/min buttons at title-bar right for FCF_MINBUTTON/FCF_MAXBUTTON (VDR-C4); `RESIZE_HIT_W=4`-px edge/corner hit zones for FCF_SIZEBORDER with `ResizeState` drag tracking (VDR-C5).
  - **VDR-D: Input routing** — `z_hit_test` dispatches mouse events to topmost frame with local-coord translation (VDR-D1); `ChromeHit` enum (`Close`/`SysMenu`/`MinButton`/`MaxButton`/`TitleBar`/`ResizeEdge`/`None`) dispatches chrome clicks (VDR-D2); `DragState { hwnd, anchor_x, anchor_y_sdl }` for title-bar drag (VDR-D3); `WM_ACTIVATE` posted on window activation (VDR-D4); `capture_hwnd` routes mouse events during resize/drag (VDR-D5/D6).
  - **VDR-E: Focus and active-window chrome** — `focused_hwnd` receives all keyboard events (VDR-E1); `WinSetActiveWindow`/`WinQueryActiveWindow` implemented (VDR-E2); focused frame title bar rendered in navy vs. inactive in dark-gray (VDR-E3).
  - 6 new unit tests in `pm_types::tests` covering Z-order operations, focus, and hit-testing.

- **PM Menu Bar Interaction** — clicking a top-level SUBMENU item in the rendered menu bar now opens a dropdown showing child items; clicking a child item posts `WM_COMMAND` (with `CMDSRC_MENU=2` source tag) to the client window's message queue and closes the dropdown:
  - **`WinDefWindowProc`** (PMWIN ord 911) — new `WM_BUTTON1DOWN` handler: if the frame has a `menu_hwnd` and the click y-coordinate is in the menu bar or a dropdown is currently open, forwards the click to `dispatch_builtin_control(menu_hwnd, WM_BUTTON1DOWN, mp1, 0)`
  - **`#Menu` WM_BUTTON1DOWN** (`dispatch_builtin_control`) — hit-tests top-level items by accumulated label width; clicking a `MIS_SUBMENU` item toggles `OS2Window::window_ulong[-1]` (0 = closed, 1-based item index when open); clicking outside any menu area closes the dropdown via `WM_MENUEND`; clicking a child item in the open dropdown computes `child_idx = (drop_y2 - 1 - click_y) / ITEM_H`, posts `WM_COMMAND{mp1 = id | (CMDSRC_MENU<<16)}` to the client window, and closes the dropdown
  - **`#Menu` WM_PAINT** — highlights the open item in navy/white; draws a 120-px-wide dropdown with 16-px-high child rows; `MIS_SEPARATOR` children render as a mid-row horizontal rule
  - **`#Menu` WM_MENUEND** — clears open state and triggers repaint
