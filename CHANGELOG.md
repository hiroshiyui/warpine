### Added
- **Sample programs** — four new OS/2 sample test applications in `samples/`:
  - `dbcs_test` — exercises `DosSetProcessCp(932)`, `DosQueryCp`, `DosQueryDBCSEnv`, `VioWrtCellStr` with a Shift-JIS DBCS pair, and `VioCheckCharType` lead/trail/SBCS classification; uses wlink `IMPORT` directive to resolve `VioCheckCharType` from VIOCALLS.DLL
  - `uconv_test` — dynamically loads UCONV.DLL via `DosLoadModule`/`DosQueryProcAddr` and exercises `UniCreateUconvObject`, `UniUconvToUcs`, `UniUconvFromUcs`, `UniMapCpToUcsCp`, `UniFreeUconvObject`
  - `audio_test` — verifies `DosBeep` (440/523/659 Hz sweep) and dynamically loads MDM.DLL to test `mciSendString` open/capability/close cycle
  - `pm_controls_test` — creates a PM frame window with WC_STATIC/WC_BUTTON/WC_ENTRYFIELD/WC_SCROLLBAR/WC_LISTBOX/WC_MLE; WM_TIMER fires after 3 s and verifies `WinQueryWindowText`, `WinSetWindowText` round-trip, `WinEnableWindow`/`WinIsWindowEnabled`, and `LM_INSERTITEM`×3 → `LM_QUERYITEMCOUNT=3`
- **`DosLoadModule`/`DosQueryProcAddr` builtin module support** — `dos_load_module` now intercepts requests for emulated modules (DOSCALLS, QUECALLS, PMWIN, PMGPI, KBDCALLS, VIOCALLS, SESMGR, NLS, MSG, MDM, UCONV) before searching the host filesystem; on first call a synthetic `LoadedDll { is_builtin: true }` is registered and its handle returned; repeat calls reuse the existing handle; `dos_query_proc_addr` detects `is_builtin` and computes the INT3 thunk address via `resolve_import(mod_name, ordinal)` instead of looking up an empty export table; eliminates `DosLoadModule('UCONV') not found on host` runtime warning; `LoadedDll::is_builtin: bool` field added to `managers.rs`; 3 new unit tests; total 416 unit tests pass
- **DBCS keyboard re-encoding (Phase B8)** — SDL2 `TextInput` event handler added to `Sdl2TextRenderer::poll_events()`; IME-composed UTF-8 text re-encoded via `cp_encode(text, active_codepage)` and pushed to `kbd_queue` as individual `KbdKeyInfo { scan: 0 }` entries; pure ASCII single-byte chars skipped (already queued by `KeyDown`); 3 new unit tests (`dbcs_text_input_cp936_encodes_to_gbk`, `dbcs_text_input_cp932_encodes_to_sjis`, `dbcs_text_input_ascii_passthrough_unchanged`); total 413 unit tests pass

- **`NlsGetDBCSEv` / `DosQueryDBCSEnv` real lead-byte table (Phase B6)** — `dos_query_dbcs_env(cb, pcc, pbuf)` shared implementation in `doscalls.rs`; writes `(first, last)` byte pairs from `dbcs_lead_ranges(active_codepage)` + `(0, 0)` terminator; returns 113 (`ERROR_BUFFER_OVERFLOW`) when `cb` too small, NO_ERROR on null `pbuf`; `NlsGetDBCSEv` handler updated from zero-stub; `DosQueryDBCSEnv` (ordinal 373) added to `api_registry.rs` and `api_trace.rs`; 5 new unit tests
- **DBCS render path in `Sdl2TextRenderer` (Phase B3)** — `render_frame()` inner loop replaced `for col` with `while col < cols` dispatching on `CellKind`; `DbcsLead` renders a 16-wide `[u8;32]` glyph (two 8-pixel halves) via `get_glyph_dbcs(ch)` and advances `col += 2`; `DbcsTail` skips (already drawn by lead); `Sbcs` unchanged 8×16 path; cursor overlay extended to 16 px for DbcsLead cells, 8 px (right half) for DbcsTail; `get_glyph_dbcs(ch) -> [u8; 32]` added as a Phase B5 placeholder (returns blank); 2 new unit tests; total 394 tests pass

### Fixed
- **VioWrtCellStr / VioCheckCharType Pascal stack corruption** — `viocalls_arg_bytes()` was missing entries for ordinal 28 (VioWrtCellStr, 5 args × 4 = 20 bytes) and ordinal 39 (VioCheckCharType, 4 args × 4 = 16 bytes); the callee-cleanup mechanism left those bytes on the guest stack after each call, corrupting local variables (e.g. `passed` counter reading as `127928197` in dbcs_test); added `28 => 20` and `39 => 16` to the match table and a VioWrtCellStr stub (returns NO_ERROR) to `handle_viocalls()`; 2 new regression tests; 418 unit tests pass
- **Far16 LSS+JMP thunk bypass** — added variant handler for `LSS ESP, [EBP-n]` prefix before `JMP FAR` (66 EA); fixes Up/Down arrow crashes in 4OS2
- **DosWaitChild 5-arg signature** — stack read used wrong offset for 4th/5th args; caused 4OS2 prompt not returning after child process exit
- **4OS2 child-process prompt regression** — relay thread and DosWaitChild interaction corrected

- **Modifier key suppression** — pure modifier keys (LShift, RShift, Ctrl, Alt, CapsLock) no longer enqueue `KbdKeyInfo` events; fixes 4OS2 printing `*`/`6` on Shift press (`text_renderer.rs`)
- **GDB Remote Stub** — `--gdb <port>` enables GDB RSP over TCP; software breakpoints, single-step, Ctrl-C interrupt, full register/memory access (`gdb_stub.rs`)
- **Crash Dump Facility** — structured crash reports on fatal VMEXITs: registers, segment descriptors, stack, code bytes at EIP, written to `warpine-crash-<pid>.txt` and stderr (`crash_dump.rs`)
- **API Call Ring Buffer** — last 256 API calls captured unconditionally for crash post-mortem (`api_ring.rs`)

- **DLL loader chain** — `DosLoadModule`/`DosQueryProcAddr`/`DosQueryModuleHandle` implemented; `jpos2dll.dll` loads at runtime with all 7 exports resolved (`lx_loader.rs`, `managers.rs`)

### Fixed
- **Far16 thunk bypass** — `#GP` on `66 EA` (Watcom `__Far16` JMP FAR) now fully bypassed in VMEXIT handler: scans stack for self-referential PUSH EBP frame, restores 32-bit caller state and returns EAX=0, avoiding all 16-bit execution (fixes 4OS2/JPOS2DLL crashes 606162, 623645, 630555)
- **SDL2 text-mode CPU idle** — replaced `poll_iter()` busy-loop with `wait_event_timeout(8)` so idle processes yield the CPU instead of pinning a full core
- Added 2048 code-tile GDT entries (GDT[4102..6149], type 0x9B) so `DosFlatToSel`/`DosSelToFlat` round-trip correctly for code-tile selectors
- Detach relay thread in `DosWaitChild` instead of joining (deadlock fix)
- Rate-limit MMIO exit handlers to prevent 100% CPU spin
- Read IDT exception frame from `SS.base+SP` when SS is 16-bit
- Relay child stdout to parent `VioManager` for `EXEC_ASYNC` `DosExecPgm`
- Detect headless mode via `is_terminal()` instead of env var
- `DosQueryAppType` uses `DriveManager` VFS for path resolution
